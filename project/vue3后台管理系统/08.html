# 使用到的组件新属性

实现表单树形结构的时候使用到了：

参数 说明 类型 可选值 默认值
row-key 行数据的 Key，用来优化 Table 的渲染；在使用 reserve-selection 功能与显示树形数据时，该属性是必填的。类型为 String 时，支持多层访问：user.info.id，但不支持 user.info[0].id，此种情况请使用 Function。 function(row) / string — —
tree-props 渲染嵌套数据的配置选项 object — { hasChildren: 'hasChildren', children: 'children' }

当 row 中包含 children 字段时，被视为树形数据。渲染树形数据时，必须要指定 row-key 。

通过指定 row 中的 hasChildren 字段来指定哪些行是包含子节点。children 与 hasChildren 都可以通过 tree-props 配置。

# 使用到的新组件：

Radio

要使用 Radio 组件，只需要设置v-model绑定变量， 选中意味着变量的值为相应 Radio label属性的值， label可以是String、Number或Boolean。

el-radio-group元素和子元素el-radio可以实现单选组， 在el-radio-group中绑定v-model，在el-radio中设置好label即可，无需再给每一个el-radio绑定变量， 另外，还提供了change事件来响应变化，它会传入一个参数value。

项目中的代码如下：

<template>
  <el-radio-group v-model="menuForm.menuType">
    <el-radio :label="1">菜单</el-radio>
    <el-radio :label="2">按钮</el-radio>
  </el-radio-group>
</template>

  this.showModal = true;
  this.action = "edit";

  this.$nextTick(() => {
    Object.assign(this.menuForm, row);
  });
},

原因：this.menuForm = row; 这样就把row的引用赋值给了 this.menuForm，在取消事件里将 this.menuForm 置空了，导致 row 也置空了，所以对应的条目内容会消失。

当使用 Object.assign(this.menuForm, row) 时，并不是将 row 的引用赋值给了 this.menuForm，当清空 this.menuForm 后并不会影响 row 。

		if (menuState) params.menuState = menuState;

	} catch (err) {

function getTreeMenu(list) {
	return list.filter((item1) => {
    // 当我们要改变从数据库中读出的数据时要通过 _doc 来操作
		item1._doc.children = [];
		list.forEach((item2) => {
			const parentIds = item2.parentId;
      // 直接取 parentIds 里面的值，取出来的是一个 object 类型，如 new ObjectId("6181ee99984be6374b19c5e3") ，需要使用 String 转成字符串
			if (String(parentIds && parentIds[parentIds.length - 1]) === item1.id) {
				item1._doc.children.push(item2);
			}
		});
    if (item1._doc.children[0] && item1._doc.children[0].menuType == 2) {
			// 快速判断按钮和菜单，用于后期做菜单按钮权限控制
			item1._doc.action = item1._doc.children;
		}
		return item1.parentId[0] === null;
	});
}
